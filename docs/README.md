# STUNN Frontend Documentation

This folder is the internal design and frontend operating system for the STUNN headless storefront.

Read in this order:

1. [`brand-system.md`](./brand-system.md) - brand positioning, palette, imagery, typography, voice.
2. [`design-rules.md`](./design-rules.md) - layout, color, type, motion, CTA, and proof rules.
3. [`pdp-philosophy.md`](./pdp-philosophy.md) - what the PDP should sell and how it should be sequenced.
4. [`component-library.md`](./component-library.md) - current architecture and reusable component map.
5. [`mobile-rules.md`](./mobile-rules.md) - mobile-first rules for PDP, cart, nav, and imagery.
6. [`do-not-do.md`](./do-not-do.md) - hard guardrails.
7. [`pdp-audit.md`](./pdp-audit.md) - current PDP critique and prioritized redesign roadmap.

## Working Rule

Do not make major UI changes from taste alone. First identify the section's job, compare it to the STUNN brand system, explain the visual direction, then implement only after approval.
